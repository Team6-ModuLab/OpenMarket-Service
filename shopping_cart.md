너는 프론트엔드 개발자야. 현재 프로젝트는 Vite 같은 번들러 없이, 순수 HTML/CSS/JS로 페이지를 구성하고 있어. “장바구니 페이지” UI를 피그마 시안과 최대한 동일하게 구현해줘.

[목표]
- 첨부한 피그마 시안(빈 장바구니 상태 1장, 상품 있는 상태 1장)과 동일한 레이아웃/간격/타이포로 구현
- 반응형: 기본은 데스크탑 기준(시안 기준)으로 맞추고, 화면이 줄어도 레이아웃이 깨지지 않게
- 기능은 최소: 수량 +/- 버튼 UI 동작, 체크박스 선택 UI, 삭제(X) 버튼 클릭 시 confirm 모달 UI 표시(실제 API 연동은 하지 않음)

[수정/생성해야 할 파일]
- HTML: <<예: pages/cart/index.html>>
- CSS:  <<예: pages/cart/cart.css 또는 pages/cart/cart-page.css>>
- JS:   <<예: pages/cart/cart.js>>
※ 현재 레포 구조를 먼저 확인해서 “장바구니 페이지가 들어갈 경로/파일명”을 가장 자연스럽게 잡아줘.
※ 기존 cart 관련 파일이 있으면 그 파일을 수정하고, 없으면 새로 생성해.

[페이지 구조]
- 상단 헤더(로고/검색창/장바구니/유저 아이콘)는 프로젝트에 이미 공통 헤더가 있으면 재사용해.
  - 공통 헤더가 없다면, 시안처럼 간단히 포함해도 됨.
- 본문은 .cart-page 같은 루트 컨테이너로 감싸서 CSS가 다른 페이지에 영향 없게 스코프를 지켜줘.

[피그마 시안 요구사항 - 공통]
- 페이지 타이틀: “장바구니”
- 테이블 헤더 영역: (체크 영역) / “상품정보” / “수량” / “상품금액”
- 전체 컨테이너는 중앙 정렬, 좌우 여백 충분히
- 카드/박스는 radius, border, 배경색을 시안처럼 부드럽게
- 폰트 크기/굵기 계층(제목/본문/보조텍스트)을 분리

[상태 1: 빈 장바구니(shopping-cart-page-none)]
- 테이블 헤더 아래 중앙에 안내 문구
  - 굵은 문구: “장바구니에 담긴 상품이 없습니다.”
  - 보조 문구: “원하는 상품을 장바구니에 담아보세요!”
- 합계 박스, 주문하기 버튼 등은 표시하지 않거나 숨김 처리

[상태 2: 상품 있는 장바구니(shopping-cart-page)]
- 장바구니 아이템 카드 리스트 UI
- 각 아이템 카드 구성:
  - 좌측: 체크박스(원형 스타일)
  - 상품 이미지 썸네일
  - 상품명(굵게), 가격, 보조 텍스트(배송/판매자 등)
  - 가운데: 수량 컨트롤( - 버튼 / 수량 / + 버튼 )
  - 우측: 상품금액(빨간색 강조), 아래 초록 버튼 “주문하기”
  - 카드 우상단: 삭제(X) 버튼
- 하단 합계 박스(회색 배경):
  - “총 상품금액” / “상품 할인” / “배송비” / “결제 예정 금액”
  - 가운데 기호( - , + )는 시안처럼 배치
  - 결제 예정 금액은 빨간색 크게
- 페이지 하단 가운데 큰 초록 버튼 “주문하기”

[인터랙션(최소 구현)]
- 수량 +/− 클릭 시 해당 카드의 수량 숫자만 변경 (최소 1)
- 체크박스 클릭 시 선택/비선택 토글(스타일 반영)
- 삭제(X) 클릭 시 confirm 모달 표시
  - 문구: “상품을 삭제하시겠습니까?”
  - 버튼: “취소”, “확인”
  - 확인 누르면 해당 카드 DOM 제거(서버 요청 없음)
- “주문하기” 버튼은 클릭 시 console.log 출력만 해도 됨                             

[아이콘/이미지 처리(피그마 추출)]
- 피그마에서 아이콘을 내가 뽑아서 링크/파일로 넣을 거야.
- 코드에 “교체 포인트”를 만들어줘(내가 쉽게 넣게):
  - 예: HTML에 <img src="<<FIGMA_ICON_PATH>>" ...> 형태
  - 또는 JS 상단에 const ICON_CLOSE = "<<...>>"; 형태
- 대상: 삭제(X) 아이콘, 체크(선택) 아이콘(또는 CSS로 구현), 헤더 아이콘 등

[개발 방식]
- CSS는 반드시 .cart-page 루트 아래로만 적용되게 작성(스코프)
- 클래스 네이밍은 BEM 또는 일관된 규칙 사용
- 빈 상태/상품 상태를 쉽게 토글 가능하게 구현:
  - mock 데이터 유무로 렌더링 분기 OR
  - 두 섹션 만들고 class로 show/hide 제어

[완료 조건]
- 시안과 최대한 동일한 UI
- 빈 상태/상품 상태 모두 구현 및 토글 가능
- 화면 줄어도 크게 무너지지 않음
- 작업 후 “어떤 파일을 어떻게 수정했는지” 변경 요약 작성
